# Project: InfinityCN

**Status:** Active | **Version:** 1.0.0 | **Type:** Cinematification Engine

## Vision
InfinityCN transforms raw novels into immersive, cinematic experiences. It prioritizes the narrative flow, emotional impact, and technical stability of the storytelling engine.

## Core Values
1. **Narrative First:** Every system must improve pacing, emotional flow, and immersion.
2. **Cinematified + Original:** Support both the transformed cinematic version and the original text version.
3. **No Monoliths:** Follow a structured pipeline: `Input -> Processing -> Structured Data -> Runtime -> UI`.
4. **Scale & Stability:** Design for large novels (100k+ words) and high user load from day one.
5. **UI Disappears:** The interface should vanish, leaving only the story.

## Success Metrics
- **Immersive Flow:** Zero-latency transitions between cinematic beats.
- **Engine Reliability:** 100% valid JSON from AI transformation or successful offline fallback.
- **Processing Scale:** Seamless handling of 100k+ word documents via server-side asynchronous jobs.
- **Offline Availability:** Full reader functionality without active network connection.

## Technology Guardrails
- **Tech Stack:** React 19, Vite, Express 5, .NET (Node/C# hybrid), Redis, RabbitMQ.
- **Persistence:** IndexedDB (Dexie) is the source of truth for the reader.
- **Design:** Modern dark-mode, glassmorphism, fluid animations (Framer Motion).

## Active Focus
Current focus is on **System Architecture**, **Backend Frameworks**, and ensuring the **Core Flow** of the application is robust and issue-free.
